# Contributing / Bidrag

Tak fordi du vil hjælpe med at forbedre DST-guiden. / Thanks for helping improve the DST register-research guide.

To måder at bidrage / Two ways to contribute:

## 1. E-mail (nemmest / easiest)

Send kode, et udkast eller bare en idé til **saras@clin.au.dk**, så lægger vi det ind. Du behøver hverken git eller GitHub.

Email code, a draft, or just an idea to **saras@clin.au.dk** and we'll add it. You need neither git nor GitHub.

## 2. Pull request

Siden er et Quarto-projekt. Hver side er en `.qmd`-fil under `docs/da/` (dansk) og `docs/en/` (engelsk), og hjemmesiden bygges automatisk via GitHub Actions.

The site is a Quarto project. Each page is a `.qmd` file under `docs/da/` (Danish) and `docs/en/` (English); the website is built automatically via GitHub Actions.

1. Fork dette repo. / Fork this repo.
2. Ret eller tilføj den relevante `.qmd`-fil. Guiden er **tosproget** - ret gerne begge sprog (eller ret det ene og skriv det i din PR, så tager vi det andet). / Edit or add the relevant `.qmd` file(s). The guide is **bilingual** - please update both languages (or do one and say so in the PR, and we'll handle the other).
3. Åbn en pull request mod `main`. Vi gennemser og fletter ind. / Open a pull request against `main`. We review and merge.

## Stil / House style

- **Ingen em-dashes** (U+2014). Brug en almindelig bindestreg med mellemrum ( - ), komma eller kolon. Gælder begge sprog. / **No em-dashes** (U+2014). Use a normal hyphen with spaces ( - ), a comma, or a colon. Applies to both languages.
- Hold `docs/da` og `docs/en` i paritet: samme overskrifter, callouts, kodeblokke, links og tabelrækker. / Keep `docs/da` and `docs/en` in parity: same headings, callouts, code blocks, links, and table rows.
- Siden bygges via GitHub Actions, ikke lokalt. / The site builds via GitHub Actions, not locally.

Spørgsmål? / Questions? **saras@clin.au.dk**
